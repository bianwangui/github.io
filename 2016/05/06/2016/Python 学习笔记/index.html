    3. 变量

    变量是物理存储单元的抽象。

    3.1 标识符（变量名）

    • 大小写敏感
    • 字母、数字、下划线开头
    • 创建有意义的变量名，避免使用字母O和l等易混淆的字符
    • 尤其在import模块时，模块名与已有变量相同会出现莫名其妙的问题

    关键字

    3.2 地址

    变量的地址是与这个变量相关联的存储地址。

    3.3 类型

    变量的类型决定了变量的取值范围及该类型值的操作集合。Python中变量类型的绑定是动态的，所有的变量都是引用，没有类型。

    3.4 值

    变量的值是与这个变量相关联的存储单元的内容。

    变量赋值的过程:

    当创建一个变量x = 2时，解释器会先找内存中有没有2的值。如果有直接做绑定，然后2这个值得引用计数加1。如果没就开辟一段内存空间，创建一个2的值; 如果修改x = 3，那么解释器就会把变量x的与3的内存地址绑定，把3的引用加1，此时，内存中的整数值2没有被引用了，当解释器检查到2的引用为0时就会启动内存回收机制把此内存空间给释放掉了。

    3.5 生存期

    静态变量、栈动态变量、显性堆动态变量、隐性堆动态变量d

    3.6 作用域

    当 Python 遇到一个变量时会按L-E-G-B的顺序进行搜索。
    全局变量只有在函数中声明为全局变量时，才能进行赋值，否则会生成一个新局部变量。
    PYTHON的作用域由def、class、lambda等语句产生，if、try、for等语句并不会产生新的作用域。

    3.7 常量

    • 常量一个重要的作用是将程序参数化。
    • None是一个特殊的常量，用来占位和变量解除绑定。

    4. 数据类型

    决定程序完成任务的难易程度的一个重要因素， 是正在使用的语言中已有的数据类型与要解决的现实问题的匹配程度。

    4.1 基本数据类型

    • 整数(大多数整数类型是硬件直接支持的)、浮点数、复数
    • 布尔类型（常在程序中用作开关或标记）

    在需要许多条件都满足才继续运行的程序中定义一个变量用于判断整个程序是否处于活动状态，这个变量被称为标志。在while语句中就只需检查一个条件（标志的当前值是否为True），并将测试是否发生了将标志设置为False的事件，从而让程序变得更为整洁。

    4.3 数组类型

    Python中的数组称为列表，它拥有动态数组的所有特点。数组的元素是对对象的引用。

    数组初始化
    数组操作

    • 赋值(Python中的数组赋值只是改变引用关系)
    • 连接（+）
    • 比较（is、==）
    • 切片（切片是把数组的一部分作为一个整体来引用的机制）
    • 判断元素是否在数组中

    4.4 关联数组

    Python的关联数组（字典）值是对对象的引用。

    如果需要搜索元素，关联数组比数组好得多，因为使用隐式散列操作来访问元素是非常有效的。另一方面，如果必须处理列表中的每个元素，则使用数组更高效。

    4.5 记录类型

    记录是指各种数据元素的集合。在集合中，单个元素通过名称来标识，并通过从结构开始的偏移量来访问（Python中无记录类型）。

    4.6 元组类型

    元组的一个用途是当必须把数组编写为受保护的类型，例如把数组发送为外部函数的一个参数，且用户不希望函数能修改参数时，就可以使用元组。

    4.7 列表类型

    Python中的列表是可变的。列表是Python的数组。

    Python中的一个强大机制，列表推导式

    4.8 指针和引用类型

    指针类型变量的取值范围是由内存地址的值和特殊的值nil构成的。nil 并不是一个有效的地址，它只表明该指针目前无法用来引用内存地址。

    • 指针提供了间接寻址的功能，这在汇编语言编程中经常用到
    • 指针提供了一种管理动态存储空间的方式
    • 指针不同于标量变量，因为它用来引用其他变量，而不是存储数据。这两种变量分别称为引用类型和值类型。

    引用类型的变量和指针区别：指针指向内存中的地址，而引用指向内存中的对象或值。在Python等面向对象语言中，所有变量都是引用。它们总是隐式地释放，而且，这些变量的值都是不能访问的。

    7. 子程序

    7.1 函数

    Python函数的def语句是可执行的。执行def语句时，其后的函数名会分配给指定的函数体。
    当函数改变其参数或者全局变量时，我们就说函数有副作用(全局变量在函数外声明，但是可以由函数访问）

    在函数中访问全局变量也是应该禁止的。但是，当对效率的要求很高时，使用全局变量来避免传递参数是提高执行速度的一种重要方法。

    参数
    非方法的子程序有两种途径访问他们要处理的数据. 第一种是直接访问非本地变量,第二种是参数传递.参数传递较直接访问本地变量更加灵活.

    参数传递
    按值传递:
    按值传递参数时,实参的值用来初始化对应的形参,然后这个实参就用作子程序中的局部变量,来实现输入语义.按值传递的缺点在于在被调用子程序内部必须为形参分配额外的存储空间.此外如果参数太大,存储空间和拷贝操作的开销会很大.
    按引用传递:
    引用传递通过访问路径获取数据的地址,并传递给被调用的子程序.引用传递传递效率较高,但对形参的访问比按值传递慢,因为引用要求额外的间接寻址操作. 如果仅需要数值单向传递给被调用的子程序,实参就可能出现无意的,错误的改动; 另一个不足之处是可以创建别名.

    Python中的参数传递方式是按赋值传递,因为所有的数据值都是对象,每个变量都是对对象的引用。在赋值传递中,实参的值赋给形参.因此,按赋值传递实际上是按引用传递，因为所有的实参值都是引用。但是，只有当对象是不可变类型时赋值传递会产生值传递语义。(纯面向对象语言)

    Python中不带圆括号的函数名就是函数的指针。函数和数据一样可以赋值给变量.因此,不需要指向子程序的指针。

    局部变量
    在子程序中定义的变量称为局部变量,因为他们的作用域通常就是所在的子程序体.

    泛型子程序
    Python的方法提供了更一般的多态性.语言中的变量没有类型,所以形参也没有类型.因此,只要定义了方法用于形参的操作符,该方法就可以使用任意类型的实参.

    Python中过程等价于函数，解释器会隐式地返回默认值None。

    在大多数面向对象编程的语言中, 当前对象的引用会隐式传送给每个方法调用.但是在Python中,这个引用必须显式传送,所以self是方法的第一个参数.

    抽象数据类型和封装结构

    封装就是为逻辑关联的软件资源（抽象数据类型）提供的语法容器。这些封装的作用是提供一种方法，将程序组织到逻辑单元中进行编译。这就允许程序的各个部分在修改后进行独立的重编译。

    面向对象程序设计的支持

    面向对象语言必须支持的三个关键语言特性：

    • 抽象数据类型
    • 继承
    • 方法调用到对应方法的动态绑定

    继承

    继承可以用来解决软件复用的问题。
    在面向对象语言中，抽象数据类型通常称为类。与抽象数据类型的实例一样，类的实列称为对象。通过对另一个类的继承来定义的类，称为派生类或子类。派生类新类的类是父类或超类。定义类对象的操作的子程序称为方法。对方法的调用通常称为消息。对象方法的完整集合称为这个对象的消息协议或消息接口。面向对象的程序中的计算，通常是通过从一个对象向另一个对象发送消息（调用方法）来说明的

    派生类与父类

    父类与子类的常见区别：

    • 父类可以定义不能被子类访问的私有变量和方法
    • 子类可以在继承父类成员的基础上再添加成员
    • 子类可以对一个或多个继承来的方法进行修改，修改后的方法与原方法有相同的名字，通常也具有与原方法相同的协议

    类可以拥有两种方法和变量。最常用的方法和变量是实例方法和实例变量。类的每个对象都拥有一套自己的实例变量，用于存储对象的状态。

    实例方法和实列变量（属性）

    实例变量仅属于类的某个对象。类变量属于类，而不属于类的对象。因此类变量对类只有一个副本。实例方法只能用于操作类的对象。类方法可以对整个类进行操作，也可以对类的对象进行操作。

    单继承与多继承

    一个子类的父类只有一个，这个派生过程叫单继承，有多个父类则叫多继承。(Python中没有接口的概念)

    继承的缺点：

    在继承层次结构中的各个类之间形成了相互依赖。这种结果正好与抽象数据类型的优点相反。抽象数据类型的优点是类型之间是相互独立的。

    动态绑定

    动态绑定是消息到方法定义的动态绑定过程所提供的一种多态性。

    并发

    软件执行时的并发的四个层次： 指令级（同时执行多条机器指令）、语句级（同时执行多条语句）、单元级（同时执行多个子程序）、程序级（同时执行多个程序）

    并发的种类

    物理并发：
    多个处理器同时运行一个程序的几个程序单元。

    逻辑并发：
    单个处理器以分时方式实现并发。

    Python程序时解释性的，所以他们只有逻辑并发。即使有多个处理器，有不能使用多处理器。

    def slowest_replace(orignal_str):
    replace_list = []
    for i, char in enumerate(orignal_str):
    c = char if char != “ “ else “-“
    replace_list.append(c)
    return “”.join(replace_list) # 92ms

    def slow_replace(orignal_str):
    replace_str = “”
    for i, char in enumerate(orignal_str):
    c = char if char != “ “ else “-“
    replace_str += c
    return replace_str # 43ms

    def re_replace(orignal_str):
    n = re.sub(‘ ‘, ‘-‘, orignal_str)
    return n # 35ms
    ​
    def compiled_replace(orignal_str):
    pattern = re.compile(“ “)
    return pattern.sub(‘-‘, orignal_str) # 29ms
    ​
    def fast_replace(orignal_str):
    return “-“.join(orignal_str.split()) # 14ms
    ​
    def fastest_replace(orignal_str):
    return orignal_str.replace(“ “, “-“) # 5ms
